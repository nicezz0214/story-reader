# Story Reader 平台规划（2026-03-23）

## 1. 当前现状

- 仓库当前基本为空，仅存在 `.gitignore`
- 适合直接按新项目方式初始化，不需要兼容遗留代码

## 2. 项目目标

建设一个“小说 / 书籍在线翻阅平台”，包含：

- 前台站点：搜索、书架、小说详情、在线阅读、翻页、移动端适配
- 后台管理：上传小说、上架、下架、删除、列表管理、基础数据分析
- 暂不建设普通用户登录系统

## 3. MVP 范围（第一阶段）

### 前台

- 首页 / 发现页
- 搜索页
- 书架页（先做“公共书架 / 最近阅读 / 收藏占位方案”，不依赖用户体系）
- 小说详情页
- 阅读页
- 移动端适配

### 后台

- 小说列表
- 小说上传
- 小说编辑
- 上架 / 下架 / 删除
- 章节管理
- 数据分析页（PV、阅读次数、热门书籍、热门章节）

## 4. 信息架构

### 前台路由建议

- `/` 首页
- `/search` 搜索
- `/shelf` 书架
- `/book/[id]` 小说详情
- `/read/[bookId]/[chapterId]` 阅读页

### 后台路由建议

- `/admin` 仪表盘
- `/admin/books` 小说管理
- `/admin/books/new` 上传小说
- `/admin/books/[id]` 编辑小说
- `/admin/books/[id]/chapters` 章节管理
- `/admin/analytics` 数据分析

## 5. 推荐技术方案（建议基线）

### 推荐基线

- 前后端一体：`Next.js + TypeScript`
- UI：`Tailwind CSS`，设计规范使用 `ui-ux-pro-max`
- 数据库：`PostgreSQL`
- ORM：`Prisma`
- 图表：`ECharts` 或 `Recharts`
- 上传方案：开发期本地存储，后续可平滑切换对象存储

### 原因

- 仓库为空，使用一个全栈应用能最快形成可运行 MVP
- 前台站点、后台管理、API、SSR 页面可统一维护
- 后续如果需要拆分独立后台或接入登录系统，仍有演进空间

## 6. UI / UX 方向（基于 ui-ux-pro-max）

### 总体风格

- 站点整体采用“极简编辑感 + 阅读器纸张感”
- 发现页 / 管理页：简洁、平面化、信息优先
- 阅读页：`E-Ink / Paper` 风格，弱装饰、低干扰、强可读性

### 视觉建议

- 主色：`#0D9488`
- 强调色：`#F97316`
- 阅读背景：`#FDFBF7`
- 正文文字：`#1A1A1A`

### 字体建议

- 标题：`Newsreader` 或 `Cormorant Garamond`
- UI 正文：`Public Sans`
- 阅读正文：优先使用高可读衬线字体

### 移动端交互原则

- 点击区域不小于 `44x44px`
- 移动端正文不小于 `16px`
- 阅读页底部提供固定操作区：上一章、目录、下一章、阅读设置
- 动效控制为轻量级，避免影响阅读沉浸感

## 7. 数据模型（MVP）

- `Book`：书名、作者、封面、简介、状态、分类、标签、字数、发布时间
- `Chapter`：所属书籍、章节标题、排序、正文、状态
- `Category`：分类
- `Tag`：标签
- `ReadingEvent`：书籍、章节、事件类型、时间、来源

## 10. 已确认关键决策记录

1. 内容导入格式
   - 推荐：先支持 `TXT / Markdown`
   - 可选：`EPUB`
   - 暂不建议首版支持：`PDF / DOCX`

2. 搜索范围
   - 推荐：先支持“书名 + 作者 + 简介 + 标签”
   - 可选：全文搜索

3. 书架定义
   - 推荐：首版做“公共书架 + 最近阅读缓存 + 收藏占位”
   - 可选：纯展示型书架，不做本地记录

4. 后台权限
   - 推荐：首版至少给后台加一个简单管理口令或基础鉴权
   - 可选：完全无鉴权，仅本地开发使用

5. 分析维度
   - 推荐：PV、书籍访问排行、章节访问排行、阅读时长估算
   - 可选：再加来源分析、时间趋势

6. 技术栈
   - 推荐：`Next.js + Prisma + PostgreSQL + Tailwind`
   - 可选：`前后端分离（Vue/React + Java/Spring）`
